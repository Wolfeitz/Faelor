# Requirements

## Functional Requirements

- The workspace must preserve project-level collaboration guidance outside world canon.
- Agents must distinguish process canon from story, world, source, visual, and rules canon.
- Repeated creator corrections must trigger a recommendation to update project guidance.
- Sensitive, constrained, private, source-limited, spoiler-limited, or undecided material must support placeholder handling.
- Agents must capture broad stubs before deep expansion after major brainstorms.
- Thing extraction must capture implied infrastructure.
- Thing stubs must support status and build depth.
- The Provocateur must review process drift as well as canon drift.

## Constraints

- All guidance must be project-agnostic.
- All lenses and workflows remain optional.
- Creator confirmation is required before durable project guidance changes unless maintenance authority is granted.
