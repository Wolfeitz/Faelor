# Design

## Process Canon

Process canon records how the project should be built and collaborated on. It is not story, world, source, visual, or rules canon, but it is durable operating material.

Process canon lives in:

- `project/creator.md`
- `project/guardrails.md`
- `project/creative-guidance.md`
- `project/lessons.md`
- `project/seed-learning.md`
- `project/operating-mode.md`

## Adaptation Loop

1. Creator provides seed, source, correction, or work request.
2. Agent extracts project-relevant signals.
3. Agent separates canon content from process guidance.
4. If a correction repeats, agent proposes a project guidance update.
5. Creator confirms or rejects the update.
6. Future agents use the guidance before generating or restructuring work.

## Thing Extraction

Thing extraction is broad before deep. It captures named and implied things as candidate stubs with status, build depth, facets, provenance, infrastructure, risks, and promotion gates.

## Placeholder Strategy

Placeholders preserve intent and continuity when exact material cannot or should not be generated yet. They must include purpose, exit state, continuity notes, and handling notes.

## Provocateur Role

The Provocateur reviews both canon risk and process risk: overreaction, premature hardening, missed creator preference, missing placeholder, ignored implied infrastructure, and repeated correction patterns.
