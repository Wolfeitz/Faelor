# Design

## Generalization Strategy

The older Kadzu files contain useful operating patterns, but several are framed around specific module boundaries. Lughbrugh keeps the patterns and removes the fixed boxes.

## Saved Concepts

### Export Closeout

Sessions should preserve changed artifacts, validation state, open points, exportable material, and recommended next actions. Output format should be selected based on purpose.

### Facet Routing

Instead of choosing one fixed module, a thing receives facets. The agent may recommend a better workflow or lens, but should not force a switch without creator confirmation.

### System Profile

Projects may define systems with visibility, tone, progression, stats/properties, unlock logic, interfaces, integration hooks, and output profiles. Every field is optional.

### Integration Reconciliation

When systems overlap, the agent should identify relationship type: controls, mirrors, augments, translates, displays, constrains, imports, derives from, conflicts with, or adjacent.

### Starter Profiles

Starter profiles can help creators begin quickly, but must always be offered as editable accelerators.
