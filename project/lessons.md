# Project Lessons

This file records process lessons learned while building this project.

Lessons here are canon-adjacent operating material. They are not world canon, story canon, rules canon, or source canon.

Use this file when a creator correction, repeated preference, tool limitation, workflow improvement, or collaboration pattern should persist across sessions.

## Lesson Format

```markdown
## Short Lesson Name

Observed:
- What happened or what the creator corrected.

Guidance:
- How future agents should behave.

Applies to:
- Files, workflows, agents, outputs, or situations.

Review:
- When this lesson should be revisited.
```

## Standing Lessons

- Preserve creator intent before optimizing structure.
- Capture broad stubs before deep expansion after major brainstorms.
- Treat repeated creator corrections as candidates for process guidance updates.
- Use the Provocateur after large brainstorms, before canon promotion, and before major structural commitments.
- Preserve useful story, gameplay, visual, or worldbuilding material even when one constrained portion needs a placeholder or later pass.
- End substantive responses with useful forward prompts: next steps, concrete actions, or focused questions. Do not make the creator drag momentum out of the agent turn by turn.
