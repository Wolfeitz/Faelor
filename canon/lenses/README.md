# Lenses

Lenses are optional advisory packs.

They may provide:

- questions
- templates
- validators
- visual directions
- export profiles
- source acquisition rules
- likely links
- prototypes and derivation rules
- archetype completeness checks
- progression/scaling rules
- manual/catalog generation patterns

Lenses must not become hidden ontology. The project may ignore, modify, combine, or remove them.

## Prototype And Derivation

When useful, a lens may define base prototypes and derived variants.

Examples:

- base goblin -> goblin shaman -> encounter instance
- base village -> mining town -> frontier boomtown
- global system message -> adventurers guild notice -> race-specific display variant
- base starship -> military cruiser -> named ship instance

Derivation should be explainable through source, progression, layers, modifiers, equipment, role changes, visual style overrides, or explicit override rationale.

Stats are optional. If the project has no stats, derivation can still track narrative role, ecology, social function, visual traits, voice, or continuity constraints.
