# Cross-Context Translation Lens

Use this lens when people, memories, abilities, rules, technology, artifacts, language, culture, or source canon move from one context into another.

This lens is optional. Do not assume a project has travel, reincarnation, portals, adaptation, imported lore, or mechanical conversion.

## Core Pattern

```text
source context -> carried element -> receiving context -> translation rule -> resulting expression
```

## Translation Types

- Direct carry: the element functions the same way.
- Reinterpretation: the receiving context maps it to a local equivalent.
- Partial carry: only some properties survive.
- Locked carry: the element is recognized but cannot be used yet.
- Dormant carry: the element exists but needs a trigger, resource, permission, or training.
- Corrupted carry: the element changes in harmful or misleading ways.
- Incompatible carry: the receiving context rejects or blocks it.
- Social carry: the actor remembers or values something, but the world has no native structure for it.

## Use Cases

- Fan fiction or series continuation with version cutoffs.
- Ruleset conversion between tabletop systems, games, or homebrew mechanics.
- Portal fantasy, reincarnation, isekai, dimensional travel, time travel, or simulation transfer.
- Adapting modern knowledge into historical, fantasy, science-fiction, or alien contexts.
- Importing a prior manuscript, campaign, wiki, database, or source corpus.
- Translating visual styles across media, factions, cultures, eras, or realms.

## Checks

- What exactly crosses the boundary?
- What is lost, renamed, blocked, or transformed?
- Does the receiving context understand intent, pattern, result, or only surface form?
- What must be learned locally before access becomes mastery?
- Who can detect the mismatch?
- What social, legal, metaphysical, mechanical, or narrative costs follow?
- Does the translation create a new base, variant, layer, or exception?

## Working Principle

Access is not comprehension. Comprehension is not integration. Integration is not mastery.

Use this principle to avoid giving an actor complete control merely because something was imported or recognized.

## Output Pattern

For each translated element, capture:

- source context
- receiving context
- carried facts
- lost facts
- local expression
- unlock or grounding requirements
- risks and contradictions
- canon status
