# System Interface Lens

Status: candidate canon
Provenance: generated from creator discussion.

## Core Model

The System has a default abstraction layer. It presents clean menus, recommended options, simplified class and skill summaries, warnings, and scenario-friendly language. Most entrants never notice this layer because it feels like fate, divine judgment, a game interface, or ordinary reality depending on expectation.

The MC recognizes it as an interface.

## Promptcraft

Promptcraft is the ability to communicate effectively with constrained intelligences, recommendation systems, AI-like interfaces, and policy-bound tools.

It cannot bypass restrictions. Its value is better inquiry design.

Useful questions include:

- List all options, including statistically disfavored options.
- Define "recommended" in this context.
- What assumptions are being used?
- Separate unavailable, locked, deprecated, dangerous, and merely disfavored options.
- What can be disclosed without violating current restrictions?
- Give the criteria categories even if exact thresholds are unavailable.
- Restate this as a decision table.
- Which parts are advisory rather than mandatory?

Core line:

> Most entrants lack awareness of their blind spots. The MC knows the first problem is learning what to ask for.

## Inner Workshop

The Inner Workshop is the MC's private System-facing learning environment. It begins as notes and query routines. Over time it can become an index, repository, translation system, simulation space, and eventually a source of shareable interface packages.

Functions:

- Capture.
- Index.
- Tag.
- Summarize.
- Cross-reference.
- Translate.
- Compare.
- Query.
- Hypothesize.
- Schedule review.
- Track confidence.
- Flag contradictions.
- Generate drills.
- Log experiments.

Knowledge states:

- Acquired: copied into the repository.
- Accessible: searchable, queryable, translatable, or summarizable.
- Comprehended: actually understood by the MC.
- Verified: tested against reality.
- Integrated: connected to body, mana, class, craft, or class-vector.
- Mastered: usable reliably under pressure.

Rule:

> He can steal knowledge, but not understanding.

Additional rule:

> Access is not comprehension. Comprehension is not integration. Integration is not mastery.

Most users see finished products. The MC learns to compose primitives into new uses.

Core line:

> The MC does not make the System do the impossible. He makes it do unintended possible things.

## Lenses And Packages

A Lens is a portable interface package that changes how a participant perceives, organizes, or trains with System information.

It grants no ability by itself. It improves feedback, comprehension, coordination, and practice efficiency.

Potential Lenses:

- Combat Overlay.
- Training Module.
- Party Tactical UI.
- Language/Translation Lens.
- Crafting/Runework Assistant.
- Personal Stat Dashboard.
- Potion Timer and Batch Logger.
- Rune Parser.
- Healer Diagnostic Lens.

They can be shared with trusted people, but recipients cannot reproduce or maintain them without the underlying System-interface skill.

## Risks

- Bad data creates bad models.
- False familiarity can be dangerous.
- Simulation quality is limited by available data.
- Lenses can overwhelm users or create dependency.
- Shareable packages may attract attention from guilds, nobles, temples, gods, or System administrators.
- Some uses may be legal under System rules but socially or religiously forbidden.
